# 振り返り運用 (retros)

## 目的
投稿後 72h 以上経過して Analytics API の値が揃った動画について、**「なぜ当たった／外れたか」を言語化** し、次回の企画・プロンプトに逆流させる。N=8〜10 の今は統計ではなく **仮説生成** のフェーズ。

## 何を書くか（大方針）
- **Measure**: `pnpm post summary` の数値を貼る（機械的）
- **Hypothesize**: 各動画の企画時の狙いと結果のギャップを言語化（人間的）
- **Learn**: 勝ち筋／負け筋の仮説を蓄積。固まってきたら生成プロンプトの few-shot に昇格させる

## 運用ルール

### タイミング
- **2〜3日に1回**、新規投稿 2〜3本が追加されたタイミングで書く
- 72h 以上経過した動画だけを「評価対象」にする（Analytics 反映を待つため）
- 72h 未満の動画も数値は貼るが「要観察」扱い

### ファイル名
`docs/retros/YYYY-MM-DD.md` （そのレビューを書いた日付）

### ワークフロー
1. `pnpm post stats` で最新データ取得
2. `pnpm post summary --sort views` と `--sort likeRate` を見比べて勝ち負けを把握
3. `cp docs/retros/_template.md docs/retros/YYYY-MM-DD.md` でテンプレ複製
4. KPI セクションは summary 出力をそのままコピペ
5. 動画別に仮説・評価を記入
6. 最後に「勝ち筋／負け筋」を更新し、次回企画に申し送る打ち手を書く

## 属性タグ（共通語彙）
仮説を束ねるために、以下の軸で毎回タグ付けする。独自軸を増やしたくなったら自由に増やしてよい。

| 軸 | 値の例 |
|---|---|
| subject | 日本史 / 世界史 / 共通 |
| era | 古代 / 奈良 / 平安 / 鎌倉 / 室町 / 戦国 / 江戸 / 明治 / 近現代 / 世界史 |
| フック型 | 疑問提示 / 否定反証(「〜は嘘」) / 意外性 / 数字インパクト / タブー裏話 |
| タイトル型 | 疑問形 / 断定＋煽り / 時間制約(「◯分で」) / 受験系 |
| 難易度 | 受験 / 雑学 / 混合 |

> メモ: これらのタグは将来 `pnpm post summary` の列として自動取得化する（別タスク）。今は手入力。

## 蓄積していくもの
- `docs/retros/YYYY-MM-DD.md` — 個別レビュー
- `docs/retros/playbook.md` — 勝ち筋／負け筋の最新版サマリ（必要になったら作る）

## 参考: KPIの見方
| 指標 | 目安（暫定） | 意味 |
|---|---|---|
| 👍率 (likes/views) | 0.5% 以上で好反応 | 内容への共感・意外性 |
| 視聴率 (avgViewPct) | 70% 以上で優秀、80%超はループ含み | フック＋構成の強さ |
| 平均視聴 (s) | 40s 超で優秀 | 同上（秒数） |
| 登録/1k | 0.5 以上で優秀 | チャンネル資産への寄与 |
| Δ24h | 24h 以内に +500 以上なら伸び続けている | 鮮度 |

※ 数値はこのプロジェクトの暫定目安。蓄積されたら更新する。
