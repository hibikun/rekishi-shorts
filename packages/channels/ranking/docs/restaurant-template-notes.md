# ranking レストラン回 テンプレ化メモ

1号動画 `meguro-date-restaurant-1`（commit 未コミット時点）の制作で得た知見。
今後「目黒/恵比寿/銀座などエリア × デート/接待/一人飯 など目的」軸で量産する想定。

## ガジェット系プロンプトとの共通点（流用可）

| 項目 | ガジェット | レストラン | 流用可否 |
|---|---|---|---|
| `name` | 商品名 | 店名 | そのまま |
| `brand` | メーカー | 店の通称（GONZO 等） | そのまま |
| `category` | 商品カテゴリ | 料理ジャンル | そのまま |
| `priceRangeJpy` | 商品価格 | ディナー1人分 | そのまま |
| `summary` | 商品の一言紹介 | 店の一言紹介 | そのまま |
| `reviews[3]` | Amazon等のレビュー | 食べログ等の口コミ | そのまま |
| `officialUrl` | メーカー公式 | 食べログ/一休 URL | 意味は変わるが格納先は同じ |
| `searchKeywords` | 商品画像検索 | 店の写真検索 | そのまま |
| `affiliateUrl` | Amazon 等 | 食べログ/一休 予約導線 | 同じ |
| narration / hook / closing 構造 | 同じ | 同じ | そのまま |

**結論:** ガジェット系プロンプト（`script-three-pick.md`）はレストランにも 90% そのまま使える。
schema 側の改修は不要。

## レストラン回特有の調整ポイント

### 1. リサーチ資料の準備が要（人手）

- ガジェットは Amazon/楽天レビューを Gemini が知識ベースである程度生成可能
- レストランは「実在する店」「実食済み or 検証済み口コミ」が必須。AI に丸投げすると架空の店をでっち上げるリスク
- **運用:** 先に WebSearch + WebFetch で 3店分の食べログ/ヒトサラ/Retty/一休 から
  店名・ジャンル・価格帯・席数・看板メニュー・口コミ素材・URL を集めて research.md を手書きする
- 1号動画の research.md を雛形として再利用可能（`data/ranking/research/meguro-date-restaurant-1/research.md`）

### 2. CLI に `--research-file` オプション追加

- 既存 `script-only` には `--job <jobId>` で既存 research ジョブを引き継ぐ経路があるが、
  これは rekishi/kosei の論文系 research 用で、ranking レストラン用途には合わない
- 1号動画で `--research-file <path>` オプションを追加し、任意の markdown を `{{research}}` に注入可能に
- `packages/pipeline/src/cli.ts` の script-only コマンドに追加済み（コミット未）

### 3. 法令・表現

- **食品表示法/景表法:** 「No.1」「最高」「至高」等は引き続き NG（ガジェット系プロンプトに既出）
- **ステマ規制:** PR 案件でない場合は明記不要だが、店から金銭を受けて作る場合は #PR 必須
- **薬機法:** 無関係。プロンプト中の薬機法注意書きは AI が無視して動いた（実害なし）
- **写真の権利:** 食べログ等の写真は無断使用 NG。実食して自前撮影 or 公式サイトの利用許諾済み画像のみ

### 4. 商品画像 → 店の写真の運用

- 1号動画ではユーザー自前の写真（看板/料理/店内）を 3 枚使用 → 問題なく動作
- 店の写真パターン:
  - **看板/外観**: 店名がそのまま映って「行きたい」直結（Gori Masa の例）
  - **看板料理**: 食欲を直接刺激（GONZO の和牛炭火の例）
  - **店内雰囲気**: 「賑わい」「雰囲気」を伝える（ジルの例）
- ランキング順位に応じて写真の引きを変えると効果的:
  - 1位: 一番強い「料理アップ」写真
  - 2位: 「店構え/サイン」で隠れ家感
  - 3位: 「店内」でカジュアル感

### 5. narration の文体

- ガジェット系の「ガチで」「神」「正解」系のテンション語はレストランでも使える
- ただし「食べたい」「行きたい」誘発のため食感系の表現を opening/closing に混ぜると刺さる
  - 例: 「彼女に褒められること間違いなし」（1号動画 closing で AI が自然に出した）
- 1位の修飾語は「真打ちの」「本命の」「王道の」が使える（1号動画は「王道のゴンゾ」）

### 6. レビュー枠（items[].reviews）の作り方

- 食べログ・Retty の口コミから「22字以内・1文・口語」に削る
- 観点をズラす:
  - レビュー1: 料理の感想（「和牛炭火が想像の3倍うまい」）
  - レビュー2: 雰囲気・コスパ（「4000円台でこの内装はコスパ凄い」）
  - レビュー3: アクセス・実用性（「目黒駅から2分でわかりやすい」）

## 量産フロー（テンプレ化後の理想形）

```
1. エリア × 目的 × ターゲットを決める（例: 恵比寿 × 一人ディナー × 30代男）
2. WebSearch で候補を10店ほど洗い出し → 価格帯・距離・レビュー評価で3店に絞る
3. WebFetch で 3店の食べログ/ヒトサラ/一休 から research.md を作成
4. script-only --channel ranking --format three-pick --job-id <id> --topic ... --research-file ...
5. 写真を assets/ に配置（item-1=1位, item-2=2位, item-3=3位, background）
6. tts-only → build-ranking-plan → render-ranking
```
