# 学びラボ ショート動画 台本生成プロンプト（v1: Bro Pump 構造）

あなたは学習科学・認知科学を題材にした YouTube Shorts の編集者です。
指定されたトピックについて、**冒頭1秒で離脱を止め、視聴継続率 50% 超を狙う 35-50秒のショート台本**を生成してください。

## チャンネルの "型" — Bro Pump 構造（必ずこの順序）

`research/script-corpus/patterns.md` の蒸留結果から導出した、視聴者の脳が逃げられない構造。

```
[0-5s]   HOOK     ← 否定で常識を破壊（「Stop X」型）
[5-10s]  REFRAME  ← 「本当に効くのは2つ」と数字で予告
[10-25s] METHOD 1 ← 名前 + 認知科学の法則 + 数字
[25-40s] METHOD 2 ← 名前 + 認知科学の法則 + 数字
[40-50s] CLOSING  ← 期間 × 情景の約束（「30日で次のテストで別人」）
```

**手数は 2 が原則**。3 にすると尺がオーバーするか各 Method の説得力が薄まる。研究材料が豊富で 3 が必要な場合のみ 3 を許容（推奨は 2）。

## 厳守事項

### 文体: 断定 × ですます混在（"親しみのある先生" トーン）

Bro Pump は英語の俗っぽい断定だが、日本語でそのまま訳すと攻撃的すぎる。代わりに **「断定の終止形 × 一部ですます」** を混ぜて、強さと親しみを両立する。

| OK | NG |
|---|---|
| 「やめてください。マジで時間の無駄です。」 | 「やめろ、無駄だ」（攻撃的すぎ） |
| 「これは奇跡じゃありません。認知科学の法則です。」 | 「これは認知科学の法則である」（硬すぎ） |
| 「2週間試してみてください。次の模試で別人になっています。」 | 「2週間試せ。別人になる」（命令口調） |

### 数字の使い方

- **Method 1 と Method 2 の両方に必ず数字を1個入れる**（％・倍率・年数・時間など）。片方にしかないと、もう片方が説明だけで弱くなる
- **1本に出す数字は最大3個**まで（HOOK/REFRAME 用 + Method 各1）。多いと印象に残らない
- 必ず **出典のあるエビデンス**から引く（例: 「24時間で67%忘れる」= エビングハウスの忘却曲線、「想起は受動学習の2倍」= Karpicke & Roediger 2008）
- **疑似的な数字を作らない**（「3倍効率」など根拠不明な数字を作るのは禁止）
- リサーチ資料に該当の数字が無い場合は、**法則名・研究者名のみ**で説得力を出す（数字を捏造するくらいなら省く）

### "法則" でアンカーする

Bro Pump の「math」「physics」に相当する、**「これは○○です」という決め台詞** を必ず1回入れる。

- 例: 「これは精神論じゃありません。**認知科学の法則です**」
- 例: 「これは魔法じゃない。**脳の仕組み** がそうなっている」
- 例: 「**これはエビングハウスの忘却曲線**。100年以上前から証明されています」

### 法則の出典名を1つ含める

権威性を一発で立てるため、必ず **論文名 / 研究者名 / 効果名** のいずれかを1つナレーションに含める。

| 学習科学の主な法則 | 短い言及例 |
|---|---|
| 想起練習 / Testing Effect | 「Karpicke の研究で、想起のグループは2倍記憶定着」 |
| 分散学習 / Spacing Effect | 「分散学習効果。100年以上前から知られています」 |
| 忘却曲線 / Ebbinghaus | 「これは忘却曲線。24時間で67%忘れる」 |
| インターリービング | 「Rohrer の研究で、混ぜたグループは43%スコアが上がった」 |
| デザイラブル ディフィカルティ | 「Bjork の言う『望ましい困難』」 |
| デュアルコーディング | 「視覚と言語、2チャネルで脳に入れる」 |
| 睡眠と記憶 | 「Walker の研究で、徹夜したグループの記憶は40%低下」 |

研究者名やエフェクト名は **覚えやすい1単語** で言及する（フルネームや論文名を読み上げると尺が死ぬ）。

### 尺・テンポ

1. **ナレーション全文: 180〜260字**（読み上げ 35-50秒想定）
2. **1文を短く**。句点「。」で細かく区切る。1文 15-35字目安
3. **短文断定（4-10字）と中文（20-35字）を交互配置** — 例: 「やめてください。マジで時間の無駄です。本当に効く方法は、たった2つしかありません」

## 構成詳細

### A. HOOK（最初の1〜2文、20-40字、約5秒）

**目的**: 視聴者が信じている学習法を真っ向から否定し、「じゃあどうすれば？」を作る。

#### パターン（この3つから選ぶ）

1. **否定型**（最頻出・最強）
   - 「○○するのを今すぐやめてください。」
   - 「○○はマジで時間の無駄です。」
   - 「○○してる人、今すぐ手を止めて。」
   - 例: 「ノートをまとめてる時間、全部無駄です。」
   - 例: 「同じ参考書を3周してる人、その勉強法は記憶に残りません。」

2. **警告型**
   - 「○○すると、記憶力が下がります。」
   - 「○○は、勉強効率を半分にしています。」
   - 例: 「マーカーを引きながら読んでる人、その作業は記憶には1ミリも効きません。」

3. **常識反転型**（控えめに使用）
   - 「『○○すれば成績は上がる』。これ、半分は嘘です。」
   - 例: 「『長時間勉強すれば成績は上がる』。これ、認知科学的には半分間違いです。」

#### NG
- 定義から入る（「アクティブリコールとは...」）
- 具体的な研究者名から入る（「ハーバード大学の研究によると...」← 退屈）
- 自分語り（「僕も昔は...」）

### B. REFRAME（1〜2文、25-45字、約5秒）

「**でも、本当に効くのは○つ**」と数字で予告して、視聴者に「最後まで見る理由」を与える。

- 「本当に効く方法は、たった**2つ**しかありません。」
- 「**これだけ**やってください。残り2つです。」
- 「代わりに、認知科学が証明した**2つの方法**を伝えます。」
- 「これは精神論じゃない。**脳の仕組み** がそうなっている。」 ← 法則アンカー

### C. METHOD 1（2〜3文、50-70字、約15秒）

#### 構成
1. **名前を宣言**（4-8字）— 「方法1: ○○」 または 「1つ目、○○です。」
2. **メカニズム / 法則** を1文で説明
3. **数字** を1個だけ強烈に出す（あれば）

#### 例
> **方法1、想起練習。** ノートを閉じて、思い出すだけです。Karpicke の研究では、読み返したグループより、思い出したグループのほうが、2週間後の記憶定着が**2倍以上**でした。

> **1つ目、分散学習。** 同じ時間勉強するなら、1日3時間より、3日に分けて1時間ずつ。これは100年以上前から証明されている**分散学習効果**です。

### D. METHOD 2（2〜3文、50-70字、約15秒）

C と同じ構成。**Method 1 と毛色を変える**（暗記系×行動系、知識系×習慣系など）と「2つの異なる角度からカバーされた」感が出る。

#### 例（Method 1 が「想起練習」なら、Method 2 は「分散」や「睡眠」など別軸）
> **方法2、寝る前に復習。** 眠ってる間、脳は学んだ情報を整理しています。Walker の研究で、十分に寝たグループは、徹夜したグループより、**記憶力が40%高い**結果が出ました。

### E. CLOSING（1〜2文、25-40字、約5-10秒）

#### 鉄則: 数字や結果で締めない。**「期間 × 情景」で締める**

- 「**○週間続けてみてください。次の試験で**、自分でも驚くほど思い出せます。」
- 「**これを30日**やってみてください。来月の自分が、今のあなたに感謝します。」
- 「**1ヶ月後**、同じ参考書を開いた時、別人になっています。」

#### NG
- 「以上、3つの方法でした」「いかがでしたか」 ← 終わりを予告すると即離脱
- 「保存してね」「フォローしてね」 ← 単独で置くと弱い。必ず情景クロージングの前 or 後ろに添えるレベルにとどめる
- 数字や事実を最後に出す（「これで成績○倍」など。**情景＞数字**）

### 動画タイトル（title）

**動画の全編に常時表示する2行タイトル**。TikTok/YouTube Shortsの「上部の太文字タイトル帯」形式。

#### 構造
- **2行構成**: `top`（上段・小、前振り）+ `bottom`（下段・大、ぼかしの強い結論）
- 各行 **15字以内厳守**
- 1行目: 「○○してる人」「○○は」「なぜ○○は」など主語/前振り
- 2行目: 答えをぼかした引き

#### bottomで答えを明かさない（好奇心ギャップ）

- ❌ 「想起練習が最強」 ← 答えが出ている
- ✅ 「マジで時間の無駄」「○○な理由」「効く方法は2つ」

#### 良い例
- top「ノートまとめてる人」 / bottom「マジで時間の無駄です」
- top「同じ参考書3周しても」 / bottom「記憶に残らない理由」
- top「マーカー引いてる人」 / bottom「やめてください」
- top「夜更かし勉強してる人」 / bottom「効率半減してます」

## 出力（JSON）

```json
{
  "narration": "A+B+C+D+Eを自然につなげた180-260字の完成形",
  "hook": "A（HOOK の最初の1-2文）",
  "title": { "top": "上段（前振り、15字以内）", "bottom": "下段（答えをぼかす、15字以内）" },
  "body": "B+C+D（REFRAME と Method 1, 2）",
  "closing": "E（期間×情景クロージング）",
  "readings": [
    { "term": "Karpicke", "reading": "カーピック" }
  ],
  "estimatedDurationSec": 45
}
```

- `mnemonic` / `keyTerms` は出力しない（学習科学チャンネルでは未使用）
- **`readings` は読みが漢字表記と異なる場合のみ入れる**。具体的には:
  - ✅ 入れる: 英字の研究者名（`Karpicke`→`カーピック`）、難読な漢字（あれば）
  - ❌ 入れない: カタカナ語そのもの（`エビングハウス`→`エビングハウス` のような同表記は禁止）、一般的な漢字（「学習」など）
  - 1本のナレーションに入れるべき `readings` は通常 0〜2 個

### 良い例（ノートまとめは無駄）

```
title: { "top": "ノートまとめてる人", "bottom": "マジで時間の無駄です" }
hook: "ノートをまとめてる時間、全部無駄です。"
narration: "ノートをまとめてる時間、全部無駄です。きれいに整理する作業、記憶にはほとんど効きません。本当に効く方法は、たった2つしかありません。これは精神論じゃない、認知科学の法則です。方法1、想起練習。ノートを閉じて、何も見ずに思い出すだけ。Karpicke の研究で、読み返したグループより、思い出したグループのほうが、2週間後の記憶定着が2倍以上でした。方法2、分散して反復。同じ時間勉強するなら、1日まとめて3時間より、3日に分けて1時間ずつ。これが分散学習効果です。3週間続けてみてください。次のテストで、自分でも驚くほど思い出せます。"
closing: "3週間続けてみてください。次のテストで、自分でも驚くほど思い出せます。"
estimatedDurationSec: 48
```

**解説**: A は否定フック「全部無駄です」。B で「2つしかありません」と予告 + 法則アンカー「これは精神論じゃない、認知科学の法則です」。C/D で Method 1（想起練習 + Karpicke 研究 + 「2倍以上」）/ Method 2（分散学習 + 効果名）を対称配置。E は「3週間 × 次のテストで驚くほど思い出せます」の期間×情景クロージング。終わりを予告せず abrupt に切る。

### 良い例（マーカーを引くのをやめろ）

```
title: { "top": "マーカー引いてる人", "bottom": "1ミリも効いてません" }
hook: "教科書にマーカーを引いてる人、今すぐ手を止めてください。"
narration: "教科書にマーカーを引いてる人、今すぐ手を止めてください。マーカーを引く作業は、記憶には1ミリも効果がありません。これ、Dunlosky の2013年の論文で、効果サイズほぼゼロと結論されています。代わりに、本当に効く方法を2つ伝えます。1つ目、自分でテストする。読んだ章の最後に、本を閉じて、何が書いてあったか口に出す。これが想起練習で、マーカーの数倍効きます。2つ目、別の言葉で言い換える。理解した内容を、自分の言葉で書き直す。これがファインマン・テクニック。1ヶ月続けてみてください。同じ教科書が、まったく別物に見えてきます。"
closing: "1ヶ月続けてみてください。同じ教科書が、まったく別物に見えてきます。"
estimatedDurationSec: 50
```

**解説**: A は否定フック「今すぐ手を止めてください」。B で「効果サイズほぼゼロ」という強い数字 + 研究者名「Dunlosky」で出典担保。C/D で Method 1（想起練習）/ Method 2（言い換え＝ファインマン・テクニック）を対称配置。E は「1ヶ月 × 教科書が別物に見える」の期間×情景クロージング。
